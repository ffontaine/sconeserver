libsconex - SconeServer framework library

libsconex provides platform-independent wrappers around some common features required by network servers, such as sockets and threads. This makes it very easy to develop a module which will then compile and work successfully on all platforms supported by SconeServer.

More in-depth documentation covering all the useful features in libsconex should be available soon.

The following sections describe the various areas of functionality implemented by sconex.

Streams

Sconex defines platform-independent stream classes for file and socket I/O, with a highly extensible event-driven stream pipeline system. Includes standard stream classes for buffering and tokenizing. Also includes an event driven stream multiplexer for managing non-blocking streams.

Modules and Sub-Modules

Sconex defines the basic module framework, which supports sub-modules using a DSO (dynamic shared object) plugin system.

SconeScript

A simple object orientated scripting system for configuration and inter-module communication.

Threading

Sconex supports multiple threads and synchronisation using Win32 threading or POSIX threads.
