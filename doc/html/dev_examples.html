Examples

SconeServer can be easily extended to handle new protocols or features by writing new modules. This section will show, using examples, how this is done.

tuesdayonly

This demonstrates the bare minimum amount of code required to implement a SconeServer module. It doesn't perform any communication - it simply decides whether to accept a connection or not depending on what day of the week it is!

rot13

This demonstrates a slightly more complex module, which deploys a stream to perform simple transforms on textual data. It also shows how connection options can be passed from the configuration files and used by the module.
